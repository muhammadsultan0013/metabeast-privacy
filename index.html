Privacy Policy — MetaBeast

Last updated: August 9, 2025

MetaBeast is a browser extension designed to generate SEO-friendly titles and keyword lists for Adobe Stock uploads. This page explains what data (if any) MetaBeast collects and how it is used.

Summary

MetaBeast does not collect, store, or share personal information such as names, email addresses, IP addresses, or other identifying details. The extension only verifies a non-personal access key to control feature access and subscription status.

Data collected

We do not collect personal data. The extension does not request or store user names, emails, locations, payment details, or other personally identifiable information.

Access key verification

To enable access control (for example, trial or paid access), MetaBeast sends a non-personal access key to a secure Supabase endpoint to validate whether the key is active. Supabase is used only to store and evaluate the access key status. No personal identifiers are stored or associated with the access key in our system.

AI processing requests

When you request title or keyword generation, the extension sends the minimal required input (for example: image metadata or a text description) to an external AI processing endpoint over HTTPS. These requests are performed solely to generate the requested metadata and are not stored on our servers by default. If any third-party AI provider logs requests according to their policies, that behavior is governed by the provider's own privacy policies.

Data storage & retention

MetaBeast itself does not store user-generated content or personal information on our servers. The Supabase backend is used only for access-key validation and does not store personal user profiles. Any temporary logs for debugging are retained for a minimal time and removed regularly.

Third-party services

MetaBeast may use third-party services for access validation and AI processing (for example, Supabase and an AI provider). We do not share or sell user data to third parties for advertising or other unrelated purposes.

Cookies and tracking

MetaBeast does not use tracking cookies or analytics for user identification. The extension may use local browser storage (via the storage permission) to save non-personal preferences like keyword-count or UI settings.

Security

All network requests to external services use HTTPS to protect data in transit. API keys and secrets are never embedded directly into the extension; server-side protection is used where required.
